# p1Finance — Product Plan

## What This Is

A personal financial research assistant for everyday investors — students and working professionals who want to invest in stocks but don't have Bloomberg or professional analyst tools.

## The Problem

Regular investors research stocks by Googling random articles, scrolling Reddit, watching YouTube, and reading scattered news. Their research is fragmented, unorganized, and has no memory. They make emotional investment decisions because they lack structured tools to build real conviction.

## The Solution

One platform to research any stock, save sources, ask AI-powered questions grounded in real data, track a watchlist with live prices, generate research reports, and visualize analytics — all in one place.

## One-Line Pitch

"Research stocks like a pro without Bloomberg — AI-powered research assistant that combines live web search, your saved sources, real market data, and analyst-grade reports in one workspace."

## Target User

- Students wanting to start investing and learn how to research
- Working professionals building wealth through stocks
- Anyone pursuing financial independence via informed investing
- People who want structured research, not scattered Googling

## Interview Positioning

Built a full-stack financial research platform using Next.js 16, Supabase, Groq, and Tavily that combines live web search, source ingestion with RAG, AI-powered analysis, real-time stock data, automated report generation, and analytics dashboards — solving the real problem of fragmented stock research for everyday investors.

## Feature Map

### 1. Dashboard — Investor Command Center

The home screen. At a glance: your watchlist with live prices, your research workspaces, and recent activity.

**What's built:**
- Workspace cards with real stats (source count, query count)
- Watchlist panel (add/remove companies)
- Dashboard stats

**What needs to be wired:**
- Live stock prices on watchlist (free API — Yahoo Finance or similar)
- Price change indicators (up/down %, green/red)
- Recent queries section across all workspaces
- Quick action: jump into any workspace or start new research

---

### 2. Workspace — The Power Screen

The main product screen. Where research happens. Chat in the center, rich side panel on the right.

**What's built:**
- Chat with Groq (working with citations)
- Live web search via Tavily (working)
- Hybrid mode: live + saved sources (working)
- Source ingestion via Tavily Extract (working)
- Sources panel with add/delete (working)

**What needs to be built:**
- **Key Metrics tab** in side panel — when workspace is tied to a company, show live stock price, P/E, EPS, market cap, volume, 52-week range. Data from a free stock API.
- **Insights tab** in side panel — AI-extracted key findings from workspace sources. Sentiment (bullish/bearish/neutral), key themes, notable risks and catalysts. Generated by Groq on demand.
- Company ticker linking on workspace creation (optional field)
- Better workspace creation flow — "What company or topic are you researching?"

---

### 3. Analytics — Research Intelligence

Real analytics generated from actual workspace data. No hardcoded charts.

**What needs to be built:**
- **Workspace selector** — pick which workspace to analyze
- **Sentiment breakdown** — Groq analyzes all sources in workspace, returns sentiment distribution (bullish/bearish/neutral with percentages). Pie or donut chart.
- **Topic extraction** — Groq identifies top recurring themes/topics from sources (e.g. "margins", "AI growth", "regulation", "earnings"). Bar chart.
- **Source timeline** — when sources were added over time. Area chart.
- **Query activity** — number of questions asked per day/week. Bar chart.
- **Key signals** — AI-extracted bullish vs bearish signals from all sources. Split view.
- All data generated on-demand from real workspace content

---

### 4. Reports — AI-Generated Research Reports

One-click report generation from workspace sources. Real structured output, not templates.

**What needs to be built:**
- **Report types:**
  - Company Summary — overview, business model, recent developments
  - Risk Memo — key risks with evidence from sources
  - Bull vs Bear Case — arguments for and against, with citations
  - Research Digest — summary of all sources in workspace
- **Generation flow:** User picks workspace + report type → Groq generates structured report from workspace sources → saved to DB → rendered as a clean document
- **Reports list** — dashboard of all generated reports, click to view
- **Report viewer** — structured sections with markdown rendering

---

### 5. Compare — Side-by-Side Analysis

Compare two companies using live search data and AI analysis.

**What needs to be built:**
- **Company picker** — select two companies to compare
- **Live data fetch** — Tavily searches for both companies, pulls recent info
- **Key metrics comparison** — side by side table (if stock data API available)
- **AI comparison summary** — Groq generates a structured comparison:
  - Business overview of each
  - Strengths and weaknesses
  - Risk comparison
  - Which looks stronger and why
- **Sentiment comparison** — how the market feels about each based on recent news

---

### 6. Landing Page — Speaks to Real Users

Rewrite to connect with the target audience.

**What needs to be updated:**
- Hero: clear value prop for students/professionals, not generic "research copilot"
- Problem/solution framing: "Your stock research is scattered. Fix it."
- Feature highlights showing actual capabilities (live search, AI analysis, reports)
- Social proof or demo preview
- Clear CTA to sign up and start researching

| Charts | Recharts |
| Deployment | Vercel |

## Architecture Principle

Every screen shows real data. No hardcoded samples. If a feature isn't ready, it's not in the nav. Every feature that ships must work end-to-end.
